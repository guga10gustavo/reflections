Quando gostaria de usar um repositório remoto em vez de manter o trabalho local?

    Para transmiter o conteudo de um repositorio para um remoto,que será usado para compatilhar o conteudo.

Por que você prefere efetuar pull de alterações manualmente do que deixar o Git atualizar-se automaticamente com o seu repositório remoto?

    Para evitar que com autalização automatica o repositorio remoto seja preenchido com commits nao desejados.

Descreva as diferenças entre bifurcações, clones e branches. Quando você usaria um e não os outros?

    .Branch são criando em um unico repositorio e sao usado como rotulos para os commits;
    .Clone é usado para criar repositorio identicos/clonar de um repositorio remoto(GitHub) para o Local ou dentro um mesmo Local;
    .Fork é usado para clonar repositorios de GitHub para GitHub.

Qual é o benefício de ter uma cópia do último estado conhecido do remoto armazenado localmente?

    O beneficio é que torna-se possivel estar atualizado sobre as possiveis atualizações feitas por colaboradores alem de possibilitar mudanças no arquivo mais recente.

Como você colaboraria sem usar Git ou GitHub? O que seria mais fácil e mais difícil?

    Eu colaboraria tentando entrar em contato com a pessoa de outra forma como por exemplo por meio de redes sociais ou e-mail.Seria mais facil a comunicação entre os envolvidos pois haveria a possibilidade de varios caminhos, ja o dificil seria a aquisição do aruivo atualizados cada um por um dos sujeitos e uni-los.
